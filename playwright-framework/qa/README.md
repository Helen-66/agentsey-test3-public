# QA 目录

## 目录用途

`qa/` 目录用于存放与质量保障（Quality Assurance）相关的文档和资源，包括但不限于：

- **测试策略与计划**：整体测试方案、回归测试策略、测试优先级等
- **测试标准与规范**：用例编写规范、缺陷分级标准、验收标准等
- **QA 流程文档**：提测流程、缺陷管理流程、发布质量门禁等
- **质量报告与度量**：阶段性质量分析报告、关键质量指标定义等

## 与其他目录的关系

| 目录 | 职责 |
|------|------|
| `tests/` | 自动化测试用例代码 |
| `qa/` | QA 流程文档、测试策略与质量标准 |
| `bugs/` | 自动化测试发现的缺陷记录（JSON，自动生成） |
| `reports/` | 测试执行报告 |

## 如何贡献

欢迎团队成员向本目录提交 QA 相关文档。请遵循以下建议：

1. 使用 Markdown 格式编写文档
2. 文件名使用小写英文加连字符（如 `test-strategy.md`）
3. 提交前请确保文档内容准确、结构清晰
